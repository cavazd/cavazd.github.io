The Dataset¶

Misinformation is a problem plaguing the everyday lives of people due to its ability to be spread on social media platforms such as Twitter. To analyze the spread of misinformation on social media, we used the dataset FakeNewsNet. This dataset created in 2018 was assembled as a part of a data collection project for fake news research at ASU. The data includes information about both BuzzFeed and PolitiFact articles, this analysis focuses solely on those from BuzzFeed. Additionally, the dataset includes information on social context related to these articles such as which articles were shared by which users, how many times an article was shared, as well as who a user follows and who follows this user.

Are there any identifiable groups in the network of Twitter followings? If so, does the spread of misinformation vary between these groups? These are the questions we are attempting to gain insight into in this analysis. To do so, we will look at some overall statistics of the dataset, find the most influential users in the network, and apply KMeans clustering to identify groups within the network.

Does the data provide sufficient evidence to suggest the mean number of times real articles are shared is different than the mean number of times fake articles are shared?

To answer this question, we can perform a 2-sample t-test at $\alpha=0.05$.

$H_0: \mu_r - \mu_f = 0$
$H_1: \mu_r - \mu_f \neq 0$
Where $\mu_r$ is the true mean number of real articles shared and $\mu_f$ is the true mean number of fake articles shared.

Find The Most Influential Users¶

To find the most influential users, first we need to store the user to user relationships in a meaningful way. To do this, we create a sparse matrix with a shape of (number of unique users x number of unique users). We can then use this matrix to calculate a modified eigenfactor score for each user. These scores are normally used to calculate the importance of scientific journals but have been tweaked slightly to calculate the importance of a user in the Twitter network.

Create matrix of Twitter connections¶

In [9]:
fin = open("BuzzFeedUserUser.txt") # open the file of user to user connections 
# save each following pair in a more friendly format (list of lists where the list is [user_id1, user_id2])
follows = []
for line in fin:
    follows.append([(int(spl.strip()) - 1) for spl in line.split('\t')]) # subtract 1 to account for 0 start indexing

# create matrix of zeros with shape (num unique users, num unique users)
mat = np.matrix(np.zeros(np.repeat(len(np.unique(follows)), 2), dtype=np.float32))
# add follows
for follow in follows:
    mat[follow[0], follow[1]] = 1  # place a 1 indicating a follow
    
# it is impossible for someone to follow themselves, so we
# go straight to standardizing
sums = mat.sum(axis=0) # find sums
# prevent divide by 0 error by inputting 1 where the sum == 0
non_zero_sums = sums.nonzero()[1]
sums = sums.tolist()[0]
for i in np.arange(len(sums)):
    if i not in non_zero_sums:
        sums[i] = 1
# standardize
for i in range(len(mat)):
    mat[i] = mat[i] / sums

# function used for eigenfactor calculations
def calc_pi_k1(alpha, mat, pi_k, dang_nodes, fol_vec):
    """
    Calculate pi^(k+1) to use in pi^* calculation.

    Parameters
    ----------
    alpha : int
        Alpha constant used for calc.
    mat : np.matrix
        Matrix used in the calc.
    pi_k : np.array
        pi^k to use for pi^(k+1) calc.
    dang_nodes : np.matrix
        Matrix of users who do not have any followers.
    fol_vec : np.array
        A list of the number of followers for each user.

    Returns
    -------
    np.array
        pi^(k+1).

    """
    left_side = (np.matmul(mat, pi_k, dtype=np.float32) * alpha).tolist()[0]
    right_side = (dang_nodes.dot(pi_k).tolist()[0][0] * alpha + (1-alpha)) * fol_vec
    return np.array(left_side) + right_side

Here we use the Fruchterman-Reingold Algorithm to determine the positions of nodes. This algorithm treats each node as a steel ring and the edges as springs connecting them. It then attempts to minimize the "spring force" between the nodes. We use this algorithm specifically to find discrete positions for our nodes to use KMeans clustering.

Perform KMeans clustering¶

Due to the visible clusters in our network, we are using KMeans clustering to identify these. The problem, however, lies in how many clusters we choose to use (k). To start, we will use some a variety of values for k.

Just from looking at the three values, we can see 10 clusters is way too many and the sweet spot for the number of clusters we want is somewhere between 2 and 5. We can reinforce this by looking at a graph of the mean Silhouette scores) for each model. The Silhouette score is a measure of cluster "goodness." A value closer to 1 means the clusters are dense and well-separated while a value closer to 0 means there is overlap with the clusters. Additionally, a value which drops into the negative could indicate points being represented by incorrect clusters.

Now that we have information about our clusters, we can begin to ask questions regarding these groupings. For example, does the data suggest the mean number of fake articles vary across the clusters? We can answer this question by using a 1-way ANOVA F-test.

Pros And Cons to This Model¶

Some potential benefits to this model are its capabilities for usage in future analysis. For example, the clusters found from this network could be paired with an analysis of the specific articles shared to attempt to predict where a user would be placed in the network. Another possibility would be to use the clusters to attempt to predict how much misinformation or information in general is shared through a specific user.

There are some harms this model could produce, however, that we should be wary of. The major problem stems from what actions companies may take if they used a model was to identify a group more likely to spread misinformation. Should they leave these users be, limit the amount of people who see their posts, or ban them from the platform altogether? These are all scenarios which could be viewed as negative depending on one's beliefs which need to be considered when applying a model such as this one. Another issue with this model is its lack of awareness of its biases. No information was provided about the specific groups of people this data was scraped from, therefore, it is hard to justify any statements made from this model as it could only be effective on a specific subset of people.

This model/analysis opens up many questions for future research. One of these is, through the use of Twitter's API, could we predict influential users based on the posts they make or could we find similarities between clusters. These are interseting questions as they could further refine our network and may expose a consistency with the spread of misinformation.
